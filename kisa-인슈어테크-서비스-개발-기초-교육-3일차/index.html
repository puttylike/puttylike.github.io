        console.log("change!!!")

console 창을 통해 e.target.value 에 input 값이 할당되는 점을 확인할 수 있다. input에 입력한 값으로 위에 텍스트가 바뀌게 코드를 수정해 보자.

e.target.value로 받아오는 거 보다는, 값을 나눠서 받아와서 세팅하도록 한다.

const handleTextChange = (e) => {
    // console.log("change!!!")
    // // 입력된 값 확인
    // console.log(e.target.value)
    // // 값 세팅
    // setmyNameWithState(e.target.value)

    // 기존 구조 나눠서 값 세팅
    const {value,name} = e.target;
    setmyNameWithState(value)

}

DB손해보험 API 소개

DB손해보험오픈API.zip 파일을 다운로드 하였다.
KISA 내에서도 특정 wifi 로만 테스트가 가능하다.

아래 경로의 jmeter.bat 파일을 실행하면 apache jmeter가 실행된다.

  • DB손해보험오픈API\Jmeter\KISA\KISA\apache-jmeter-5.1.1\bin

apache jMeter는 postman과 유사한 프로그램이다.

테스트 하기 좋게 어느 정도 프로그래밍이 되어 있다.

  • open을 클릭해 backups 위치에 있는 해외여행자보험-000048.jmx 를 선택한다.

인증 방식에서는 Oauth 방식을 활용한다.

벅스 사이트 로그인을 예로 들면, 직접 벅스 id를 쓸 수도 있지만, facebook이 인증을 할 수도 있다.
: 자원 소유자의 본인 인증 => 인증되면, client에 요청해 accessToken을 발급 => 자원서버 => 자원 소유자

User Defined Variables 를 수정하고, 이용기관 접큰키를 선택하고 Ctrl + T 한 뒤 start를 누르면
키 값을 받을 수 있다. (Access Token)
= 이를 로그인으로 보면 된다.

같은 거를 Postman으로도 요청해 본다.
: Post를 쓰며, Request Header 값을 그대로 Header에 입력하면 된다.

그럼에도 jMeter를 쓰는 이유는, 보험료 산출 로직이 그 안에 있어서다.

[단체]API마스터증권 ~ 에 회사에서 받은 마스터증권을 넣고 저장한 뒤,
Ctrl + T 와 Start를 차례로 하면 Response를 받을 수 있다.
