1. Pandas

ML이 아닌, 데이터 분석(상관계수, 표준편차 등)에 대해 알아 보려면 pandas 공부를 해야 한다.

cf. 책<파이썬 라이브러리를 활용한 데이터 분석> by 김영근(pandas 핵심 멤버)

  • 데이터 분석은 데이터 학습 (패턴 분석) 위주여서 (분포/상관관계/공분산 등) current status 에 집중하기에, ML처럼 예측하는 건 덜 하다.

  • pandas ?

    • Dataframe : 2차원 배열, 테이블 형태이다.
      행렬과 달리 문자형이 있다. 여러 데이터 타입을 가질 수 있다.
      1개 이상의 series(열 하나)로 구성돼 있다. 인덱스와 행 번호가 따로 있다.

cf. anaconda를 설치했다면, pandas는 기본적으로 설치돼 있다.

> pip list
pandas 0.23.0

  • read_csv(…)

    • 가장 자주 쓸 듯 싶다.
    • 왼쪽에 0부터 인덱스이고, (자동으로) 파일 내용 맨 위는 열 이름으로 세팅된다.

df['Job'] => 열 데이터 추출 방법은 이외에도 많다. ​

  • 행 기준 추출

    • iloc 순수한 행 순서 슬라이싱
    • loc 인덱스 바꿀 수 있음 => 혼선에 주의할 것. 기존 파이썬 slicing과 혼선이 없으려면 iloc를 쓸 것을 권장한다.

df.iloc[ -1] df.loc[-1] => err 발생 : KeyError: 'the label [-1] is not in the [index]'

cf. NaN = null = missing data 결측치
=> 내버려두면 분석이 안 된다 ML에서 쓰려면 사전에 반드시 처리해야 한다

  • null 체크 방법
    .isnull.sum() => null 갯수 확인 .value_counts() => 데이터 분포를 확인할 수 있다. ​
  • null 있는 모든 행 제거
    df.dropna()
    => 데이터가 통으로 없어지는 단점이 있으니, => fillna() 로 값을 변경하자.
  • fillna()
    () 안에 옵션으로 , inplace=True 설정해 줘야 실제 파일 내용이 교체된다.

숫자 컬럼 빈 칸 채울 때, 강제로 보통 평균이나 중간값으로 채운다.
=> 오류가 발생할 수 있다만, 이 점은 감안해야 한다.

ex. 0,0,0,100,100 => 평균 40 중간값 50

Kaggle - Titanic Data

numpy는 행렬이니, pandas로 string과 null에 대한 전처리 이후 numpy로 사용해야 한다.

​pd.read_csv() => type : dataframe (df) 이다.
=> 여기 df에 df.values하면, type : numpy.ndarray numpy이다...!
=> 열심히 전처리하고 마지막에 .values만 붙이면 바로 numpy 완성된다.

  • Titanic data는 str 이 들어 있기 때문에 numpy.loadtxt 안 된다
    => pd.read_csv 로 불러오고 전처리해서 df.values 로 numpy 만들어 DL하는 순으로 진행 해야 한다.
  • .head() : 앞에 5개가 나온다. (.head(2) 등 가능하다.)
  • 모든 입력 data 11개 가 생존에 기여하는가?
    관련 없는 건 분석에 활용할 필요가 없다.
    다만, 필요한 data가 null이면 fillna로 채워야 한다.
  • cabin 굳이 drop 안 하고 .values 했더니 행렬 element에 문자열이 들어감 => 사전에 숫자 변환 필요하다. => 문자를 숫자로 바꿀 땐 규칙성만 있게 바꿔주면 된다.
