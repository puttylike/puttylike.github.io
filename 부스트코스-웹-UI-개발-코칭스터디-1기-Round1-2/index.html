3. 콘텐츠모델, 시멘틱마크업, 블록 & 인라인

(1) 콘텐츠 모델

HTML5에는 요소들이 가지고 있는 성격에 따라 요소의 종류를 정의하는 규칙들이 존재한다. => 콘텐츠 모델

  • Content Models 의 7 분류

    • Metadata Content :
      " base, link, meta, noscript, script, style, title " 콘텐츠의 스타일, 동작을 설정하거나 다른 문서와의 관계 등 정보를 포함하는 요소들이 포함된다. 메타 태그, 타이틀 태그, 스타일 태그, 링크 태그가 이에 해당하며 대부분 내에 들어간다는 것이 특징이다.
    • Flow Content :
      " a, abbr, address, map>area, article, aside, audio, b, bdo, blockquote, br, button, canvas, cite, code, datalist, del, details, dfn, div, dl, em, embed, fieldset, figure, footer, form, h1 ~ h6, header, hgroup, hr, i, iframe, img, input, ins, kbd, keygen, label, map, mark, math, menu, meter, nav, noscript, object, ol, output, p, pre, progress, q, ruby, samp, script, section, select, small, span, strong, style[scoped], sub, sup, svg, table, textarea, time, ul, var, video, wbr " 문서의 자연스러운 흐름에 의해 배치되는 요소들이 포함되며, Metadata에 해당하는 일부 태그들만 Flow에서 제외되며 요소 대부분이 Flow에 포함된다.
    • Sectioning Content :
      " article, aside, nav, section "
      HTML5에서 새로 생긴 <article>, <aside>, <nav>, <section> 등이 포함되며 이 태그들은 문서의 구조, 아웃라인에 영향을 주게 된다.
    • Heading Content :
      " h1, h2, h3, h4, h5, h6 " 각 section의 header를 정의하는 heading 태그가 포함된다.
    • Phrasing Content :
      "a, abbr, map>area, audio, b, bdo, br, button, canvas, cite, code, datalist, del, dfn, em, embed, i, iframe, img, input, ins, kbd, keygen, label, map, mark, math, meter, noscript, object, output, progress, q, ruby, samp, script, select, small, span, strong, sub, sup, svg, textarea, time, var, video, wbr" 문서의 텍스트 또는 텍스트를 꾸며주는 문단 내부 레벨로 사용되는 요소들이 포함된다.
    • Embedded Content :
      " audio, canvas, embed, iframe, img, math, object, svg, video " 외부 콘텐츠를 표현하는 요소들이 포함되며 오디오나 비디오, 이미지 등 멀티미디어 관련 요소들이 이에 해당된다.
    • Interacitve Content :
      " a, audio[controls], button, details, embed, iframe, img[usemap], input, keygen, label, menu, object[usemap], select, textarea, video[controls] " 사용자와 상호작용을 하는 요소들이 포함되며 대표적으로 form 요소들이 해당된다.

(2) 시멘틱 마크업

  • 시멘틱 마크업(POSH(Plain Old Semantic HTML)) : 의미론적인 HTML, 컴퓨터(브라우저)가 잘 이해할 수 있는 코드

    • 의미에 맞는 요소를 사용
    • 문서의 구조화
    • 인간과 기계가 모두 이해할 수 있는 것이 목표
  • 시멘틱 마크업 하기

ex. 중요하다는 의미를 포함할 때는 <b>가 아닌 <strong>을 사용하는 것이 적절하고 시멘틱한 마크업이다.

<b>굵은</b> vs <strong>중요한</strong>
<i>기울어진</i> vs <em>강조하는</em>
<u>밑줄친</u> vs <ins>새롭게 추가된</ins>
<s>중간선이 있는</s> vs <del>삭제된</del>

(4) 블록&인라인

요소들을 총 7개의 콘텐츠 모델로 구분하기 전에는 크게 블록 레벨과 인라인 레벨로 구분했다. 시각적으로 차이가 아주 명확하기 때문이다.

  • block level 요소 : 한 줄에 하나의 요소 표시
    " div, h1~h6, p, ul, li, table ..."
    => 일반적인 모든 요소(블록, 인라인 레벨 등)를 포함할 수 있다.
  • inline level 요소 : 한 줄에 여러 개의 요소 표시
    " span, i, img, em, strong, a ..."
    => 블록 레벨 요소가 인라인 레벨의 자손으로 들어가는 것은 허용하지 않는다. (인라인 레벨 요소는 블록 레벨 요소를 감쌀 수 없다.)

cf. HTML5 버전에서 생겨난 한가지 예외 경우가 있는데, <a>는 인라인 레벨 요소지만 어떠한 블록 레벨 요소도 자손으로 가질 수 있다.

4. CSS 이해하기

(1) CSS 소개

  • CSS (Cascading Style Sheets) : HTML (마크업 언어)를 꾸며주는 언어로, html이 웹페이지의 정보를 표현한다면, CSS는 html을 보기 좋게 디자인하는 역할을 한다.
    => CSS는 보통 마크업 언어인 HTML과 같이 묶여 언급된다.
    (Style Sheets는 스타일 규칙을 정의하는 것을 말한다..)

cf. http://www.csszengarden.com

웹사이트에서 자주 사용되는 HTML 태그는 10여개 이지만, CSS 덕분에 다양한 디자인을 갖게 되었다.

(2) CSS 문법과 적용

  • CSS 구문 
h1 { color: yellow; font-size:2em; }
/* CSS 주석 */
  • 선택자(selector) - "h1" => 어느 요소를 꾸밀 것인지
  • 속성(property) - "color"
  • 값(value) - "yellow"
  • 선언(declaration) - "color: yellow", "font-size: 2em" => 선언은, 속성:값; 으로 표현하며, 선언과 선언 사이 개행 가능하다. (속성이름과 속성값 사이에는 개행 불가.)
  • 선언부(declaration block) - "{ color: yellow; font-size:2em; }" => 마지막 선언에는 ;가 필요없다.
  • 규칙(rule set) - "h1 { color: yellow; font-size:2em; }" => CSS 여러 개의 규칙으로 이뤄져 있다.

cf. HTML - 속성(attribute) / CSS - 속성(property)

  • CSS 적용 방식

    • inline (해당 요소에 직접 스타일 속성을 이용해 규칙 선언)
    • internal (head tag 내 style tage 안에 스타일 규칙 추가)
    • external (외부 스타일 시트 파일(.css 형식의 css 파일) 활용, head tag 내 link tag 안에 선언 후 파일 경로 작성 필요(css 파일 연결 시 rel="stylesheet" 필요), 파일 관리가 편하면서도 용량이 작아 주로 사용되는 방법이다.)
    • import (스타일 시트 내에서 다른 스타일 시트 파일을 불러오는 방식, 내부 상단이나 외부 스타일 시트 파일 상단에 선언하는데 성능상 좋지 않아 거의 안 쓰인다.) 
/* 1. inline */
<div style="color:red;"> 내용 </div>

/* 2. Internal */
<style> div {color: red;} </style>

/* 3. external */
/* .css */
div {color: red;}
/* .html */
<link rel="stylesheet" href="css/style.css">

/* 4. import */
@import url("css/style.css");

(4) 선택자 2

요소/전체 선택자만으로는 (선택) 사용에 한계가 있다. => class/id 선택자

  • class 선택자
    html global 요소 중 class를 활용한다.

HTML을 수정해 class 속성을 추가한 뒤, CSS에서 그 값을 선택자로 지정해 사용한다. 클래스 선택자를 쓸 때는, 맨 앞에 .(마침표)를 찍어야 한다. 

.foo { font-size: 30px; }
<p class="foo"> ... </p>
  • 다중 class
    하나의 요소에 여러 개의 class를 넣을 수도 있다.
  • id 선택자
    . 대신 #(해쉬 기호)을 쓰면 된다.

  • class 선택자와의 차이점

    • .기호가 아닌 #기호 사용
    • 태그의 class 속성이 아닌 id 속성을 참조
    • 문서 내에 유일한 요소에 사용
    • 구체성
  • 클래스 선택자는 여러 요소에 같은 클래스를 넣고 같은 규칙을 적용 할 수 있지만, class와 달리 id는 문서 내에서 유일해야 한다. 그리고 구체성의 값이 다르다.

cf. Difference between id and clss

(5) 선택자 3

  • 선택자를 조합하는 방식

    • 요소 + 클래스 : p.bar => p, bar 둘다 있어야 적용
    • 다중 클래스 : .foo .bar => foo bar 둘다 있어야 적용
    • id와 class의 조합 : #foo .bar => d가 foo이며 class가 bar여야 적용 
/* 요소와 class의 조합 */
p.bar { ... }

/* 다중 class */
.foo.bar { ... }

/* id와 class의 조합 */
#foo.bar { ... }

  • 속성 선택자 = 태그의 속성을 이용해 선택

    • 단순 속성으로 선택
    • 정확한 속성값으로 선택
    • 부분 속성값으로 선택

  • 단순 속성으로 선택 

    p[class] { color: silver; }
p[class][id] { text-decoration: underline; }
    <p class="foo">Hello</p>
<p class="bar">CSS</p>
<p class="baz" id="title">HTML</p>

    => 속성의 값은 상관하지 않고, class, class/id 속성만 존재하면 적용된다.

  • 정확한 속성값으로 선택 

    p[class="foo"] { color: silver; }
p[id="title"] { text-decoration: underline; }

    => p 이면서 해당 class / id 값에 해당하면 적용된다.

  • 부분 속성값으로 선택 : 속성 이름과 속성값 사이에 사용되는 기호에 따라 다르다.

    • [class~="bar"] : class 속성의 값이 공백으로 구분한 "bar" 단어가 포함되는 요소 선택
    • [class^="bar"] : class 속성의 값이 "bar"로 시작하는 요소 선택
    • [class$="bar"] : class 속성의 값이 "bar"로 끝나는 요소 선택
    • [class*="bar"] : class 속성의 값이 "bar" 문자가 포함되는 요소 선택

(6) 문서 구조 관련 선택자

구조는 실제로 문서의 스타일이 적용되는데 훨씬 많은 역할을 한다.
선택자와 문서 사이의 관계를 이해하기 위해 어떻게 문서가 구조화되는 지 알아야 한다.

  • 문서 구조의 이해
    부모 요소는 그 요소를 포함하는 가장 가까운 상위 요소로, 그 요소의 부모 요소는 단 하나뿐인 반면, 자식 요소는 부모 요소와 반대라고 생각하면 되며 자식 요소는 여러 개일 수도 있다.
    ex) 의 부모 요소: ↔ 의 자식 요소:
  • 조상과 자손
    조상 요소는 그 요소를 포함하는 모든 요소로, 부모 요소를 포함하여 여러 개일 수도 있으나, 자손 요소는 그 반대로, 그 요소가 포함하고 있는 모든 요소가 자손 요소이다.
  • 형제
    같은 부모를 가지고 있는 요소들은 서로 형제 관계에 있다. 형제 관계에 있는 요소 중 바로 뒤에 이어 나오는 요소는 인접한 관계로 본다.

문서 구조를 이용한 선택자는 3가지가 있다.

  • 문서 구조 관련 선택자

    • 자손 선택자
    • 자식 선택자
    • 인접 선택자
  • 자손 선택자
    아무 기호없이 그냥 공백으로 구분.
  • 자식 선택자
    선택자 사이에 닫는 꺽쇠 기호(>)를 넣어 표현.
  • 인접 형제 선택자
    선택자 사이에 + 기호를 사용. 
/*자손 선택자 : <div>의 자손 요소인 <span>를 선택하는 선택자*/
div span { color: red; }

/*자식 선택자 : <div>의 자식 요소인 <h1>를 선택하는 선택자*/
div > h1 { color: red; }

/*인접 형제 선택자 : 형제 관계이면서 바로 뒤에 인접해 있는 요소를 선택하는 선택자*/
div + p { color: red; }

/* body 요소의 자식인 div 요소의 자손인 table 요소 바로 뒤에 인접한 ul 요소 선택! */
body > div table + ul { ... }

(7) 가상 선택자 1

  • 가상 선택자(pseudo selector)
    지금 문서 내에 존재하지 않는 요소에 스타일을 부여하거나
    특정 요소의 상태를 미리 추정해서 가상의 클래스로 스타일을 적용시킬 수도 있는 선택자.

    • 가상 클래스(pseudo class)
    • 가상 요소(pseudo element)
  • 가상 클래스
    미리 정의해놓은 상황에 적용되도록 약속된 보이지 않는 클래스
    (브라우저가 특정 상황이 되면 자동으로 클래스를 적용시킴)

ex.

HTML과 CSS는 정적인 언어이기 때문에
동적인 변화가 필요할 때는 다른 언어를 사용해야 하는데, 이는 개발 비용이 들어가는 일이다.
그래서 CSS에서는 흔하게 사용되는 여러 패턴에 대해서 미리 정의해놓고, 가상 클래스로 제어할 수 있게 했다. 

:pseudo-class {
    property: value;
}

가상 클래스는 :(콜론) 기호를 써서 나타낸다. 가상 클래스를 이용하면 CSS만으로 처리가 가능하므로 훨씬 효율적이다.

cf. 가장 기초적이고 대표적인 가상 클래스

  • 문서 구조와 관련된 가상 클래스

    • :first-child : 첫 번째 자식 요소 선택
    • :last-child : 마지막 자식 요소 선택

  • 앵커 요소와 관련된 가상 클래스

    • :link : 하이퍼 링크이면서 아직 방문하지 않은 앵커
    • :visited : 이미 방문한 하이퍼링크를 의미
      cf. 하이퍼 링크는 앵커 요소에 href 속성이 있는 것을 의미한다.

  • 사용자 동작과 관련된 가상 클래스
    에 주로 쓰인다만, 조건에 맞는 상황이 되는 요소들은 다 사용이 가능하다.

    • :focus: 현재 입력 초점을 가진 요소에 적용
    • :hover: 마우스 포인터가 있는 요소에 적용
    • :active: 사용자 입력으로 활성화된 요소에 적용

(9) 구체성

같은 요소를 선택하는 서로 다른 규칙들이 상반된 스타일을 가지고 있다면?
선택자에 어떤 규칙이 우선으로 적용되어야 하는지에 대해 정해진 규칙에 의해 적용된다.

  • 구체성(명시도)
    구체성은 선택자를 얼마나 명시적으로(구체적으로) 선언했느냐를 수치화한 것으로, 구체성의 값이 클수록 우선으로 적용된다.

구체성은 4개의 숫자 값으로 이루어져 있는데, 좌측에 있는 값부터 비교하며, 좌측 부분의 숫자가 클수록 높은 구체성을 갖는다.

[0, 0, 0, 0]

  • 0, 1, 0, 0 : 선택자에 있는 모든 id 속성값
  • 0, 0, 1, 0 : 선택자에 있는 모든 class 속성값, 기타 속성, 가상 클래스
  • 0, 0, 0, 1 : 선택자에 있는 모든 요소, 가상 요소
  • 전체 선택자는 0, 0, 0, 0을 가진다.
  • 조합자는 구체성에 영향을 주지 않는다. (>, + 등) 
h1 { ... }  /* 0,0,0,1 */
body h1 { ... }  /* 0,0,0,2 */
.grape { ... }  /* 0,0,1,0 */
*.bright { ... }  /* 0,0,1,0 */
p.bright em.dark { ... }  /* 0,0,2,2 */
#page { ... }  /* 0,1,0,0 */
div#page { ... }  /* 0,1,0,1 */
  • 인라인 스타일

인라인 스타일의 구체성 값은 1, 0, 0, 0이며 규칙들 중 가장 큰 구체성을 갖는다.

p#page { color: red; }
<p id="page" style="color:blue">Lorem impusm dolor sit.</p>

=>

에는 color: blue가 적용된다.

  • !important
    별도의 구체성 값은 없지만, 모든 구체성을 무시하고 우선권을 갖는다. 속성값 뒤 한 칸 공백을 주고 느낌표 기호와 함께 쓴다. 
p#page { color: red !important; }
<p id="page" style="color:blue">Lorem impusm dolor sit.</p>

=> important 때문에 color red가 적용된다.

cf. (Cascade and inheritance)[https://developer.mozilla.org/en-US/docs/Learn/CSS/Buildingblocks/Cascadeand_inheritance]

(10) 상속

  • 상속
    구체성과 더불어 선언들이 문서에 어떻게 적용되는지 이해하기 위한 또다른 핵심 개념이다.
    특정 요소만이 아니라 그 자손 요소에까지 적용된다.

cf. margin, padding, background, border 등 박스모델 속성들은 다 상속되지 않는다. 상속되는 속성들은 보통 상식적으로 구분될 만한 속성들이다.

  • 상속되는 속성의 구체성 
* { color: red; }
h1#page { color: gray; }
<h1 id="page">Hello, <em>CSS</em></h1>

=> 상속된 속성은 아무런 구체성을 가지지 못하기 때문에 에는 red가 적용된다.

cf. (Inheritance)[https://developer.mozilla.org/en-US/docs/Web/CSS/inheritance]
