예제 3

와인 품질(wine quality)을 가상의 bad, good 두가지 타입으로 예측

quality_bins = [ 3, 6.7, 10]   # [3, 6.7) 1구간, [6.7, 10) 2구간
quality_label = [ 'bad', 'good']  # 3부터 6.7 구간은 bad, 6.7부터 10구간은 good

wine_scaled_df['quality'] = pd.cut(wine_scaled_df['quality'],
                                   right=False, include_lowest=True,
                                   bins=quality_bins, labels=quality_label,ordered=False)

quality 를 bad, good 으로 분하는데,
분류 기준은 quality.mean()이 5.8, quality.std() 0.88 정도이므로 6.7 을 기준으로 bad, good 등으로 나눴다.

right=False, include_lowest=True 3<=x<6.7
6.7<=x<10

cf. right=False, include_lowest=True로 쓰거나,
right=True, include_lowest=False로 써야 에러가 안 난다. (구간 겹쳐서 error)

그런데 right=True, include_lowest=False로 쓰면,
3<x<=6.7
6.7<x<=10
인데, 정답은 3~9이다. 이 경우,
3이 포함 안 되기 때문에, right=False, include_lowest=True로 썼다.

이렇게 쓰고 싶다면, bins를 바꿔야 한다.

quality_bins = [ 2, 6.7, 10]   # [3, 6.7) 1구간, [6.7, 10) 2구간
quality_label = [ 'bad', 'good']  # 3부터 6.7 구간은 bad, 6.7부터 10구간은 good

wine_scaled_df['quality'] = pd.cut(wine_scaled_df['quality'],
                                   right=True, include_lowest=False,
                                   bins=quality_bins, labels=quality_label,ordered=False)

2<x<=6.7

예제 4

와인 품질(wine quality)을 가상의 bad, good, best 세가지 타입으로 예측

quality_bins = [ 3, 5, 8, 10]   # [3, 5) 1구간, [5, 8) 2구간, [8, 10) 3구간
quality_label = [ 'bad', 'good', 'best']  # 3부터 5 구간은 bad, 5부터 8구간은 good, 8부터 10구간은 best

wine_scaled_df['quality'] = pd.cut(wine_scaled_df['quality'],
                                   right=False, include_lowest=True,
                                   bins=quality_bins, labels=quality_label,ordered=False)

quality 를 bad, good, best 로 분류하는데,
분류기준은 평균이 5.8, 표준편차 0.8 이므로
bad 기준은 mean - std = 4 까지, good 기준은 mean + std = 5.8 + 0.8 = 6.6, 즉 7 까지, best 는 8 이상으로 해보았다.

CNN

딥러닝은 2012년, 컴퓨터 비전은 2015년을 기점으로 나뉜다.

2012년, Alexnet이라는 CNN이 나오면서 image 에 대한 정확도가 올라갔다.
2015년, Yolo가 나왔다. 그 이전에는 Fast R-CNN이 사용되고 있었다.

데이터가 정해지면 입, 출력층은 고정이다.
아키텍처 관점에서 은닉층만 변화된다고 볼 수 있다.

Convolution

시간 t의 흐름에 따른 필터(커널)가 stride 이동하면서 원본 데이터 변화의 평균
스트라이드의 스탭을 시간으로 생각해도 될 듯...

𝑓 (𝜏) * 𝑔 (𝑡) = ∫ 𝑓 ( 𝜏 )𝑔 (𝑡 - 𝜏) 𝑑𝜏

딥러닝은 loss가 최소화되는 w, b를 찾는 거 였다.
CNN에서는 그 filter인 g(𝑡 - 𝜏)를 찾는 게 목적이다.

𝑓 ( 𝜏 ), 𝑔 (𝑡 - 𝜏) : numpy
이미지 : 행렬(numpy)
