AI 이노베이션 스퀘어 12기 기본반 1일차 후기

cf. AI 이노베이션 스퀘어 사이트 : https://ai.koipa.or.kr/

AI 이노베이션 스퀘어 기본반에 지난 8월 접수 신청하여 교육생으로 선정됐다. 사실, 되게 예전부터 듣고 싶었는데, 입사 초부터 정신 없었고, 그 당시는 아마 평일만 강의가 있었지 싶고, 주말에도 일해서 체력적으로 불가했다. 그 사이 주말반도 생겼고, 수업(강의반) 개수도 더 많이 늘었고, 나도 시간적 여유가 생겨서 이번 기수에야 신청할 수 있었다.

조금 아쉬운 점이 있다면, 코로나여서 오프라인이 아닌 온라인으로 강의를 진행한다고 한다. 그럼에도 이 하나의 아쉬운 점 때문에 신청하지 않는 건 스스로에게 너무 핑계대는 거 같아서... 신청했다.

이 강의는 박성호 교수님과 함께 한다. 기존 수강생 후기에 대학 교수님이 가르치신대서 신청한 게 크다. 예전에 학원에서 배웠던 게 그닥 도움이 되지 않았어서. 좀더 원론적인 강의가 듣고 싶었는데, 박 교수님이 지향하는 부분이 그러해서 좋았다. 저작권 문제가 있으니 강의 자료는 올리지 않는다. 자료 제공 문의도 받지 않을 예정이다.

1. 개요

금일 강의는 OT+개요+파이썬 기초 순으로 진행됐다. 이 강의는 python + jupyter notebook을 쓴다. 뭘 쓰든 상관없다. 다만, 파이썬 api가 디폴트인 게 많을 뿐... ㅎㅎ

(1) 4차 산업혁명 : 키워드는 연결, 인공지능!

(2) 인공지능 ⊃ 머신러닝 ⊃ 딥러닝 인공지능이랑 머신러닝이 다른 건 알고 있었는데, 인공지능이면서 머신러닝이 아닌 게 뭔지는 몰랐었다. 대표적을 퍼지 이론이 있다고.

(3) 머신러닝 ML - 회귀 + 분류 ... 하지만 이 ML에도 한계가 있어, 딥러닝 DL이 탄생했다 한다.

(4) 딥러닝 DL - 신경망, 뇌 세포 상호작용을 모방했다고.

(5) 더하기에는 평균이, 곱하기에는 원본에 대한 변환이 들어간다는 insight...! 오늘 가장 충격 받은 부분. 내가 대학교 전공이 수학과여서 더 그랬나 싶다.

(6) 어지간한 data는 이미지여서, 행렬을 쓴다. 이유도 이제 알다니, 역시 과거의 나는 헛배웠다. 이렇게 명확하게 개념을 재정립할 수 있어서 이 강의가 너무 마음에 든다.

(7) 파이썬에서 중요한 데이터 타입은, list / string / tuple / dictionary / set

2. 파이썬 데이터 타입

(1) 리스트 : 연속적 데이터 저장용 []. 값은 인덱스로 가져온다.
파이썬에서는 마이너스인덱스를 지원하여 리스트의 마지막부터 역순으로 값을 참조할 수 있음

cf. 리스트 안에 리스트는... C의 구조체 기능 덕분에 가능하다고.
리스트에서 제일 중요한 건 슬라이싱 : 기능이다.

그리고 append! 특정 수치에 대한 값 변화를 계속 append해서 그래프 만들 때 유용하다는 insight...!

(2) 튜플 : readonly list () cf. 파이썬 함수 return 값 > tuple

형태 >>> 딕셔너리, set

(3) 딕셔너리 {} : hash map 스타일의 데이터 타입
key:value => 하나의 행 >> 인덱스 말고 키 값을 사용한다.
입력한 순서대로 데이터가 들어가는 것이 아님. 순서대로 append 해보자.
json 가공 최소화할 때 형태의 유사성 때문에 dictionary를 쓰자는 insight...!

(4) set : 데이터가 중복되지 않는다
라벨링 시 차집합 매우 유용하다는 insight...!

(5) string : '' 또는 ""를 사용해서 생성하는데, 단, 시작과 끝이 같아야 함.
split() / strip() 함수는 머신러닝 코드에서 문자열 데이터 전처리(pre-process) 하기 위해 자주 사용
cf. split의 return 값은 리스트.
