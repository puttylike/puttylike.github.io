11차시) 송금

1. 송금의 역사

금융의 기초, 송금.

  • 송금의 역사 : 은행의 대면서비스에서 ATM기기로 발전했고, 텔레뱅킹 서비스에서 모바일 뱅킹, 오픈뱅킹까지 발전을 거듭했다.

과거에 송금을 하기 위해서는 계좌이체 신청서, 통장, 신분증이 필요했다.
=> 편리해진 건, 텔레 뱅킹 서비스가 가능해져서다. (은행코드 / 계좌번호 / 보안카드 필요)
=> 이후, 모바일 뱅킹의 시대가 도래했다. : kB국민은행 - 세계 최초의 모바일 뱅킹 서비스 뱅크온 출시
=> 2009년 11월, 하나은행 - 하나N뱅킹 출시
=> 6년 뒤인 2015년, 비바리퍼블리카, 토스 app 출시 (공인인증서 / OTP / 계좌번호 필요 X)
=> 카카오페이 - 대화창에서도 송금 서비스 개시

한 은행에서 다른 은행으로 돈을 보내는 것 = 송금 (금융결제원의 금융공동망 사용함)
=> 스타트업 / 빅테크의 고민 : 수수료 => 오픈뱅킹이 해결

2. 해외 송금 서비스

해외에 있는 사람에게 돈을 송금하거나 받는 경우? 주거래은행 > 송금+수수료 > 중계은행 > 송금+수수료 > 해외은행 > 수취+수수료
: 과거 해외송금은 많은 중개단계와 높은 수수료가 당연했다.

=> 달러는 달러끼리 하면 안 되나?
=> 2011년 영국, TransferWise : 송금수수료를 낮추고, 환전수수료 X
: 2011년 영국의 트랜스퍼와이즈는 해외송금의 복잡한 절차와 높은 수수료를 개선한 대표적인 서비스
=> 현재 1일 최대 95만원, 1년 최대 5만달러까지(개인) OK

  • 카카오뱅크 : 해외송금 서비스 개시
    5천만달러 이하
    5천원 초과시 수수료 10000원
    "빠른 송금 / 적은 수수료 / 해외 계좌 검증"

TransferWise 외국회사, 언어와 신뢰의 장벽 존재.
=> 국내 은행, 저렴한 서비스 등장

카카오페이 로 해외 송금이 안 될까?
=> 페이스북 리브라 프로젝트 !

  • 리브라의 구조
    cf. 24억/15억/10억 (페북/왓츠앱/인스타 이용자수)
    고객(송금인) -> 전자지갑 -> 리브라 (블록체인) -> 전자지갑 -> 상점(수신인)

리브라 전자지갑(칼리브라 등) / 리브라 블록체인 / 준비금(주요국 통화 은행예금 및 단기유가 증권 등)

cf. 출처 : 금융위 리브라 이해 및 관련 동향

  • 토스의 성장 배경
  • 송금의 고질적인 문제였던 개인인증의 문제를 혁신적으로 개선한 서비스로 2021년 7월 제3의 인터넷 전문은행 출범 예정

= 금융결제망 사용 비용 때문. (400~500원 정도)

=> 오픈뱅킹 등장 : 금융결제망을 전면 개방하는 사업
= 400~500원 수수료 => 40~50원 (토스, 카카오페이 연락처, 친구 송금/환전 수수료 0원)

모든 송금이 스마트폰이 있어야만 할까?
없어도 된다 : 케냐 M-PESA > 휴대폰 번호 = 계좌 번호
=> 엠페사 계정 충전

핀테크를 넘어선 따뜻한 금융, 빠른 송금이 아니라 쉽고 친절한 송금

12차시) 결제

1. 금융결제의 변화

교통카드는 현금으로만 충전이 가능하다.
길거리 노점이나 5천원 이하으 물건을 사면서 카드를 내는게 민망해서 지나칠 때가 있다.

버스, 지하철 / 식사, 커피 / 택시 => 돈을 쓰는 일은 일상에서 매순간 일어나는 일!

플라스틱 카드 / 삼성페이 가 있어 편리하게 결제할 수 있게 됐다.
모바일 결제를 넘어선 금융 결제.

2. 해외 모바일 결제 서비스

cf. 페이팔 Mafia
회사를 만들었던 주역들이 지금도 IT기업들을 이끌고 있기엔 붙여진 이름
: 피터 앤드리아스 틸 / 일론 리브 머스크 / 스티브 챈 / 채드 헐리 = TESLA / LINKED IN / Palantir / Yelp / Youtube / Yammer 수뇌부

  • 페이팔 = 미국의 간편 결제 서비스
    1999년 콘피니티, X.COM이 합병하며 Paypal 탄생, 2002년 이베이가 인수한 회사
    => 구매자 > Paypal에 지불 > 판매자에 송금
    = 거래 상대방에게 자신의 은행 계좌번호나 신용카드 번호의 노출없이 클릭 한 번으로 송금 처리를 완료할 수 있는 서비스 제공

= PDA 팜 파일럿 가진 사람끼리 거래 => E메일로 송금할 수 있게 서비스 확장 => 이베이의 판매자들이 페이팔을 이용해 거래 => 폭발적 성장
=> 이베이가 인수하게 된 계기

"페이팔 : 핀테크의 시작"

  • 팜 파일럿 단말기 => 스마트폰

cf. 구글 페이 (G Pay) : NFC 활용하여 스마트폰으로 결제
= 교통카드 역할 / 송금 가능
= 국내 사용 불가
= 구글이 개발한 전자지갑 플랫폼의 하나로, 모바일 기기에서 인앱 및 탭투페이(tap-to-pay) 구매를 지원

cf. 애플 페이 : 2014년 출시, 40개국 서비스 - 아이폰 Wallet 앱에서 사용 가능 / NFC
= 국내 사용 불가
= 애플이 제공하는 모바일 결제 및 전자 지갑 서비스
=> 이후 실물 신용카드 출시 (카드번호, CVC 번호 없음) : NFC 결제 불가 대체용

모바일 결제, QR 코드의 천국은? 중국 = 위챗페이 / 알리페이

  • 알리페이 : 알리바바 그룹 산하 시스템 - 에스크로 서비스(페이팔과 유사, 구매 확정 시 판매자에게 돈 송금) / 전기세 등 공과금 납부
    = 중국의 서드파티 모바일 및 온라인 지급 플랫폼
  • 위챗페이 : 중국 사람들가의 대화/사업을 위해서는 위챗 사용 > cf. 샤오청쉬 : 기업의 서비스를 이용하기 위해 앱을 설치할 필요 없이 미니앱으로 결제 가능
    = 텐센트가 개발한 인스턴트 메신저 위챗을 통한 모바일 결제 서비스
  • 우리나라 = 신용카드 보편화
    중국 = 신용카드 20% => 사람이 많아 측정이 어려워서 현글/직불카드 보편화

3. 삼성페이와 국내 모바일 결제 서비스

삼성페이 = 2015년, 삼성전자에서 공개한 간편결제 서비스로,
MTS 마그네틱 방식 결제를 보유한 루프페이를 인수해 삼성페이 탄생
= 삼성페이는 마그네틱 보안전송(MTS)을 사용하여 무선으로 마그네틱 신용카드의 정보를 전송해서 결제하는 방식으로, 근거리 무선 통신(NFC)을 사용하지 않아도 됨

cf. 구글페이/애플페이 는 NFC 기반
cf. 국내 매장에는 NFC 단말기 부재, 대부분 마그네틱 방식으로 결제

삼성페이가 급속도로 퍼져나갈 수 있었던 이유? 카드사들에게 수수료를 받지 않음
=> 삼성 스마트폰 락인효과
cf. 2020년 하반기 해외 결제를 지원하기 시작하면서 카드사들과 수수료 협상

LG페이 = 삼성페이와 유사

카카오페이 = 2014년부터 서비스, 2018년부터 오프라인 결제 서비스 시작, 모바일 강점

네이버페이 = 온라인 강점

cf. 네이버페이 에 제로페이 얹어 결제 가능

=> 간편 결제들은 온/오프를 넘나들며 서비스 확장 => 빅데이터를 통해 맞춤형 서비스 제공

2019년 말, 네이버 - 네이버 파이낸셜 분사로 금융서비스 확대

제로페이 = 2018년, 전국에 확산되면서 사용자 증가
= 대한민국 정부에서 소상공인 결제 수수료를 거의 줄이기 위해 금융 회사, 결제 회사와 공동으로 시행하는 모바일 간편 결제 서비스

=> 은행은 온/오프 결제가 될까?
=> 제로페이를 연동해 두었다면 제로페이 결제가 가능한 곳에서 은행앱을 실행해 QR을 인식하면 즉시 제로페이로 결제가 된다.

=> 데이터로 합쳐서 제대로 분석될 때, 맞춤형 서비스 !
