ex) 추천 사이트 w3schools => Learn CSS 클릭

  • CSS Reference를 통해 확인 가능한 사항들

    • 정의 : 해당 속성이 어떤 변화를 일으키고 어떻게 동작하는지 알 수 있다.
      기본 값 / 상속 여부 / 애니메이션 가능 여부 / 사용 가능한 CSS버전
      cf. CSS 버전에 따라 브라우저 지원율이 달라진다.
    • 문법 : 해당 속성값을 어떤 식으로 나열하여 사용하는지를 파악할 수 있다.
    • 속성 값 : 해당 속성이 인식하여 적용할 수 있는 값의 형태나, 키워드 등을 파악할 수 있다. Initial : 초기값, 즉 속성의 기본값으로 정의 (ie에서 지원하지 않음) Inherit : 부모 요소의 해당 속성 값을 적용 (상속 가능할 요소일 경우) 즉, 상속이 불가능한 속성일 경우에는 inherit을 지정해도 적용되지 않는다.
    • 지원 범위 : 해당 속성이 정의에 맞게 동작 가능한 CSS 버전, 브라우저별 버전을 확인할 수 있다. 어떤 브라우저의 어떤 버전이냐에 따라 같은 값이어도 다르게 렌더링 될 수 있으므로, 현재 프로젝트의 사용자 제공 지원 범위를 잘 확인하고 적용해야 한다.
    • 예제 : 문법과 속성값을 바탕으로 실제로 속성을 동작하는 예제 코드를 확인할 수 있다.
    • 참고 사항 : 해당 속성에 대해 특이사항이나 버그에 대해서 확인할 수 있다.

ex) 추천 사이트 MDN => 기술목록 CSS => 완전한참고서 클릭

2) 속성-단위

CSS 단위 절대 길이 px ( 1px = 1/96th of 1 inch )
웹에서 가장 많이 사용되는 단위로 (1px = 화면 1개의 점) 컴퓨터 화면 등 장치의 해상도에 따라 상대적인 단위이다. 웹 개발 시 여러 환경에서도 디자인을 동일하게 표현하고, 브라우저 호환성에 유리한 장점을 가지고 있어 디자인 의도가 많이 반영된 웹사이트에서 픽셀 단위 사용을 권장한다.
=> 상대 단위지만, 화면에서 고정된 크기 값을 가지기에 절대 길이이다.
pt ( 1pt - 1/72 of 1 inch )
컴퓨터가 없던 시절부터 있던 단위로, (인쇄물이나 워드프로세서 프로그램에서 사용된 가장 작은 표준 인쇄단위) 웹 화면에 인쇄용 문서를 위한 스타일을 적용할 때 유용하다. 사용하는 기기의 해상도에 따라 차이가 있어 (Windows에서는 9pt = 12px, Mac에서는 9pt = 9px) W3C에서도 pt는 웹개발 시 권장하는 단위가 아니다.
=> 디자이너가 pt로 주면 같은 값의 px로 개발한다.

상대 길이
% : 부모의 값에 대해서 백분율로 환산한 크기를 갖는다.
기본 글꼴 크기에 비해 상대적인 값을 갖는다. 상위 요소에 정의된 픽셀을 %로 늘이거나 줄일 수 있다.
=> ex. font-size 100% 상위 요소 16px => font 16px

em : font-size를 기준으로 값을 환산합니다. 소수점 3자리까지 표현 가능합니다.
해당 폰트 대문자 M의 너비를 기준으로 하는 상대적인 단위다. 1em은 현재 지정된 폰트 크기와 같다. (2em은 200%)

rem : root의 font-size를 기준으로 값을 환산합니다. vw : viewport의 width값을 기준으로 1%의 값으로 계산됩니다.

3) 속성-색상

color 속성은 폰트의 색상 값을 적용할 때 사용한다.
cf. color 속성 뿐만 아니라 색상과 관련된 다른 속성들에도 적용된다.

  • 다양한 색상 값의 적용법

    • 컬러 키워드 - ex) red (cf. transparent는 투명을 나타내는 키워드)
    • 16 진법 : 6자리의 16진수(0-9, A-F)는 각각 두 자리씩 세 가지 색상을 나타낸다. ex) #RRGGBB (#ffffff => white / #ff0000 => red)
    • 16 진법 : 각각의 두 자리가 같은 값을 가지면 3자리로 축약하여 사용할 수 있다. ex) #RGB
    • RGB( ) : 0 → 255는 검정 → 흰색으로 값의 변화를 나타낸다. (RGB(255,0,0) =>red)
    • RGBA( ) : A는 0 → 1은 투명 → 불투명으로 값의 변화를 나타낸다. (RGBA(255,0,0) => 취소선으로 적용되지 않는다. A까지할 거면 꼭 하나 더 써 줘야 한다. RGBA(255,0,0,0.5) => 반투명한 red) 
body{color:black} /* by color names */
body{color:#000000} /* Hexadecimal colors */
body{color:#000} /* Hexadecimal colors */
body{color:rgb(0,0,0)} /* AS RGB values (red, green, blue / 0 ~ 255) */
body{color:rgba(0,0,0,1)} /* (red, green, blue, alpha /  0 ~ 255, 0 or 1) */
/* AS HSL values (CSS3) */
/* AS HWB values (CSS4) */

4) 속성-background

background 관련 속성
background bg-color bg-image position/bg-size bg-repeat bg-origin bg-dip bg-attachment initial|inherit;

  • background 축약
    background: [-color] [-image] [-repeat] [-attachment] [-position];

background 속성은 요소 배경을 컬러나 이미지로 적용 시 사용한다. 

/* 속성 */
{background-color: green;
background-image: url("img_tree.gif");
background-repeat: no-repeat;
background-position: top center;
background-attachment: scroll;
}
/* 축약형 */
{background: green url("img_tree.gif") no-repeat fixed center;}

cf. https://www.w3schools.com/css/css_background.asp

cf.

는 블록 요소라 갖는 영역은 전체 콘텐츠 영역 만큼
cf. 은 인라인 요소이기 때문에 갖는 영역은 텍스트 길이 만큼

  • background-color (기본 값 : transparent)
  • background-image (기본 값 : none)
  • background-repeat (기본 값 : repeat)
    repeat : x, y축 으로 모두 반복
    repeat-x : x 축 방향으로만 반복
    repeat-y : y 축 방향으로만 반복
    no-repeat : 이미지를 반복하지 않음
  • background-position (기본 값 : 기본 값 : 0% 0% 요소) 값의 선언 순서는 x축, y축으로부터의 간격이다. 만일 한쪽만 지정된다면 나머지는 중앙 값(center)으로 적용된다.
    % : 기준으로부터 % 만큼 떨어진 지점과 이미지의 % 지점이 일치하는 곳에 위치
    px : 기준으로부터 px 만큼 떨어진 지점과 이미지의 (0,0) 지점이 일치하는 곳에 위치
    키워드 : top, left, right, bottom, center 키워드를 사용 가능.
    키워드는 선언 순서와 관계없이 top, bottom은 y축 기준으로 하며 left, right는 x축을 기준으로 한다.
  • background-attachment (기본 값 : scroll) 화면 스크롤에 따른 배경 이미지의 움직임 여부를 지정하는 속성 scroll : 배경 이미지는 요소 자체를 기준으로 고정되어 있으며 내용과 함께 스크롤 되지 않음.
    local : 배경 이미지는 요소의 내용을 기준으로 고정되어 있으며 내용과 함께 스크롤된다.
    fixed : 배경 이미지는 뷰포트를 기준으로 고정되어 있으며 스크롤에 영향을 받지 않는다.
    cf. 뷰포트 : 사용자가 시각적으로 볼 수 있는 웹페이지 영역 (컴퓨터나 휴대폰과 같은 장치에 Display 요소가 표현되는 영역)

5) 속성-boxmodel

HTML 모든 요소는 사각형의 박스로 만들어진다.

  • boxmodel 구성
    (1) Content 영역 : 요소의 실제 내용을 포함하는 영역. => 크기는 내용의 너비 및 높이를 나타낸다.
    (2) Border 영역 : content 영역을 감싸는 테두리 선
    (3) Padding 영역 : content 영역과 테두리 사이의 여백. content 영역이 배경, 색 또는 이미지가 있을 때 패딩 영역까지 영향을 미친다. => padding을 content의 연장으로 볼 수도 있다.
    (4) Margin 영역 : border 바깥쪽의 영역. border 영역을 다른 요소와 구별하기 위해 쓰이는 빈 영역이다. = 주변 요소와의 여백(간격)을 margin을 이용해 지정할 수 있습니다.

6) 속성-border

  • border : 요소의 테두리
  • border 축약
    border: [-width] [-style] [-color];
/* 속성 */
border-width : 1px; /* 두께 <length> | thin | medium | thick */
border-style : solid; /* 종류 none | hidden | dotted | dashed | solid | double | groove | ridge */

border-width: 6px; /* 보더 두께 */
border-width: 6px 3px; /* 보더 상하 좌우 두께 */
border-width: 6px 3px 4px; /* 보더 상, 우(좌) 하 두께 */
border-width: 6px 3px 4px 1px; /* 보더 상, 우, 하, 좌 두께 */

7) 속성-padding

  • padding : 요소의 안쪽 여백 (기본값 : 0)

  • 속성값

    • length : 고정값으로 지정합니다. (ex. px, em ....)
    • percent : 요소의 width에 상대적인 크기를 지정
    • padding-top : content 영역의 위쪽 여백을 지정
    • padding-right : content 영역의 오른쪽 여백을 지정
    • padding-bottom : content 영역의 아래쪽 여백을 지정
    • padding-left : content 영역의 왼쪽 여백을 지정

  • syntax
    속성의 순서는 위쪽을 기준으로 시계방향으로 돌아가는 게 고정이다.
    축약형으로 사용할 때 반드시 4개의 속성에 대한 값을 모두 적어야 하는 것은 아니다. 

    padding: [-top] [-right] [-bottom] [-left];
      0      10px     20px      30px   /* 상, 우, 하, 좌 다름 */
      0      10px     20px           /* 좌, 우 같음 */
      0      10px                 /* 상, 하 같음 & 좌, 우 같음 */
      0                         /* 상, 우, 하, 좌 모두 같음 */

cf. CSS에서 0 값에 대해서는 단위를 따로 적지 않는다.

cf. padding

8) 속성-margin

  • margin : 요소의 바깥 여백 (기본값:0)

  • 속성값
    (1) length : 고정값으로 지정합니다. (ex. px, em ....)
    (2) percent : 요소의 width에 상대적인 크기를 지정
    (3) auto : 브라우저에 의해 계산된 값이 적용

    • margin-top border 영역의 위쪽 여백을 지정
    • margin-right border 영역의 오른쪽 여백을 지정
    • margin-bottom border 영역의 아래쪽 여백을 지정
    • margin-left border 영역의 왼쪽 여백을 지정 
{
margin-top: 0;
margin-right:100px;
margin-bottom: 20px;
margin-left: 30px;
}

{
margin:0 10px 20px 30px; /* 상,우,하,좌 다름 */
margin:0 10px 20px; /* 좌우 같음 */
margin:0 10px; /*상,하 같음 좌,우 같음 */
margin:0; /*상,우,하,좌 같음 */
}

{
margin-left: auto; /* 브라우저 계산 */
margin: auto; /* 가로축 중앙 정렬 (요소에 width 값 필요) */
}

  • syntax 

    margin: [-top] [-right] [-bottom] [-left];
    0      10px     20px      30px   /* 상, 우, 하, 좌 다름 */
    0      10px     20px            /* 좌, 우 같음 */
    0      10px                   /* 상, 하 같음 & 좌, 우 같음 */
    0                            /* 상, 우, 하, 좌 모두 같음 */

cf. margin auto 기본적으로 브라우저에 의해 계산이 이루어지는데, 대부분의 경우 0(기본값) 또는 요소의 해당 측면에서 사용 가능한 공간과 같은 값을 갖는다.

  • margin collapse(마진 병합) 인접한 두 개 이상의 수직 방향 박스의 마진이 하나로 합쳐지는 것
  • 마진 병합이 일어나는 경우
    (1) 두 요소가 상하로 인접한 경우: 위 요소의 하단 마진과 아래 요소의 상단 마진의 병합이 일어납니다.
    (2) 부모 요소와 첫 번째 자식 요소 또는 마지막 자식 요소 부모 요소의 상단 마진과 첫 번째 자식 요소의 상단 마진 병합이 일어납니다.
    (3) 부모 요소의 하단 마진과 마지막 자식 요소의 하단 마진 병합이 일어납니다. 내용이 없는 빈 요소의 경우: 해당 요소의 상단 마진과 하단 마진의 병합이 일어납니다.

마진 병합은 수직 방향으로만 이루어지며, 좌우에 대해서는 일어나지 않습니다. 마진 병합은 마진이 반드시 맞닿아야 발생하기 때문에 2,3번째의 경우 패딩 및 보더가 없어야 합니다. 마진 병합을 활용하여 첫 번째와 두 번째 컴포넌트의 조합이 다양한 경우 여백을 다르게 사용 할 수 있습니다.

cf. (Mastering margin collapsing)[https://developer.mozilla.org/en-US/docs/Web/CSS/CSSBoxModel/Masteringmargincollapsing]
(css margin)[https://www.w3schools.com/css/css_margin.asp]

9) 속성-margin&padding

  • margin과 padding의 비교

      • auto 단위

        margin o o o px, % ...
        padding o x x px, % ...

margin은 음수 값 적용이 가능하고, padding은 적용되지 않는다.
%는 상하좌우의 방향에 관계없이 모두 요소의 width 값을 기준으로 값이 결정된다.

10) 속성-width

  • width : 요소의 가로 크기 (기본 값 : 0)
  • 속성값
    auto : 브라우저에 의해 자동으로 계산하여 적용합니다. * 요소의 레벨 기본 특성에 따라 다르게 동작합니다.
    length : 고정값으로 지정합니다. (ex. px, em ....)
    percent : 부모 요소의 width에 상대적인 크기를 지정합니다. 
width: auto; /* 기본값 */
width: 100px; /* 길이 */
width: 50%; /* 퍼센트 */

width는 padding, border 영역에 대해서 영향을 받는다.

cf. width

11) 속성-height

  • height : 요소의 세로 크기 (기본 값 : 0)
    contents 요소 내용의 높이 값 선언 시 사용.
  • 속성값
    (1) auto : 브라우저 자동으로 계산하여 적용합니다. * 기본적으로 컨텐츠 영역의 내용만큼 높이를 가집니다.
    (2) length : 고정값으로 지정합니다. (ex. px, em ....)
    (3) percent : 부모 요소의 height에 상대적인 크기를 지정한다.
    cf. 부모 요소가 명시적으로 height 값이 있어야 한다. 
height: auto; /* 기본값 */
height: 100px; /* 길이 */
height: 50%; /* 퍼센트 */

디자인 가이드가 올 때를 대비해 계산할 줄 알아야 한다.
ex) 너비 160px => 100px content + ( 10px * 2) padding + ( 15px * 2 ) border = 150px

% : Containing Block의 높이 값
parent와 cild 사이에 child가 차지할 수 있는 박스 영역
=> 부모가 명시적인 높이 값을 가지고 있어야 자식 높이를 % 으로 지정할 때 적용이 된다.
cf. https://developer.mozilla.org/en-US/docs/Web/CSS/height#values

cf. height 또한 box-sizing 속성을 이용하여 기준값을 padding 영역, border 영역으로 바꿀 수 있다.

cf. height

12) 속성-boxmodel정리

box model - contents / padding / border / margin
content 영역의 너비는 width, 높이는 height을 통해서 선언할 수 있다.
전체 box sizing 의 값 = 요소의 전체 크기 = contents 영역의 width, height에 padding 값과 border 값을 더한 값
다른 요소와의 여백은 margin으로 선언할 수 있고, margin은 음수값으로도 선언 가능하다.
margin은 상하 요소 사이의 병합 현상이 일어나고 이때는 큰 값을 기준으로 병합된다.
