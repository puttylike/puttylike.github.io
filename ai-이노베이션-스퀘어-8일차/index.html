​(2) Logistic Regression Algorithm

분포를 나타내는 직선을 찾아야 한다 = W, b를 찾아야 한다

=> $z = W*x+b$ : 직선은 $-∞$ 과 $-∞$ 사이의 값을 갖는데, 0, 1만 필요해서 여기서는 sigmoid 함수 활용한다. 그런데 sigmoid는 0과 1사이 값이 나오니까 변곡점 0.5 기준으로 값을 구분할 것이다.
= 어떤 입력이 들어와도 출력은 0과 1사이로 압축해서 보여준다.

cf. 정규화는 시스템에서 입력에 대해 처리하는 것,
cf. softmax는 출력을 확률로 표현하는 것,
cf. data distortion 데이터 왜곡

  • 활성화 함수 activation function

ex) sigmoid : $y = 1/(1+e^(-x))$

ex) reLu : $y = max(0, x)$

ex) In RNN : $tanh = ( e^x - e^(-x) ) /( e^x + e^(-x) )$

​* 바닥부터 동작원리 이해할 땐 sigmoid를, 실무에서는 속도 때문에 reLu를 쓰긴 한다. 대용량 아닌 이상 sigmoid 써도 무방하지만.

=> $y = sigmoid(z)$ 에서 0~1 사이 값이 나오므로,
연속 값을 갖는 선형회귀 때와는 다른 손실함수가 필요해 진다.

  • sigmoid 0~1 나오니까 확률로 해석(?)... cross entropy 쓰면 확률이 높아진다.

(3) cross entropy 유도 : 확률분포로 식을 유도한다

$E(W,b) = -log(L(W,b)) = -Σ{t_i * log(y_i) + (1-t_i) * log(1-y_i)}$

cf. $L(W,b)$ -> 우도함수로, 입력 x에 대해 정답 t가 발생될 확률

log 계산 시간이 오래 걸려서 속도를 중시하면, linear regression의 손실함수 E를 쓰기도 한다. MSE나 cross entropy나 뭘 쓰든 상관은 없다.

(4) 구조

Linear Regression과 다른 점은 손실함수 계산 부분이다.
LR에선 $y = W*X + b$ 뿐이었다면, Logistic Regression에서는 Regresson 이후 $z = sigmoid(y)$까지 해야 한다는 점이다.
