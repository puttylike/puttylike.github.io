Github Blog 만들기2 - table Of Contents 추가(by gatsby)

다른 블로그 돌아다니다 보니, 게시글에 대한 목차가 있는 게 뭔가 신기했다. 마크다운 문법 찾으며 정리하던 중에 이를 TOC 라고 하는 걸 알게 됐다. 내가 나중에 글 다시 읽을 때 유용할 거 같아 보여서 블로그에 해당 기능(?)을 추가하기로 마음 먹었다.

gatsby-remark-table-of-contents 플러그인이 존재하지만, 추석 기념으로 수정해 봤다. 리액트 공부를 해야 이걸 이해할 텐데... ㅎㅎ... ㅠ...
[참고] mdx 말고 markdownRemark(md)를 사용하며, 최소한의 소스코드만 사용합니다.

깃허브로 블로그 만들기 - 2

(1) 관련 플러그인 설치

  1. gatsby-plugin-mdx 설치
$ npm install gatsby-plugin-mdx @mdx-js/mdx @mdx-js/react
  1. gatsby-config.js 파일 수정 - plugins 안에 추가한다.
module.exports = {
  plugins: [
    {
      resolve: `gatsby-source-filesystem`,
      options: {
        name: `posts`,
        path: `${__dirname}/src/posts/`,
      },
    },
    {
      resolve: `gatsby-plugin-mdx`,
      options: {
        extensions: [`.mdx`, `.md`],
        gatsbyRemarkPlugins: [],
      },
    },
  ],
}
  1. localhost:8000/__graphql 에서 allMdx 를 쓸 수 있게 된다.
{
  allMdx {
    nodes {
      excerpt
    }
  }
}
  1. gatsby-remark-auto-headers 설치
$ npm install gatsby-remark-auto-headers
  1. gatsby-config.js 파일 수정 - gatsby-transformer-remark 안에 추가한다.

gatsby-remark-prismjs를 쓰고 있다면, gatsby-remark-auto-headers 이 먼저 오게 써야 한다.

(2) toc 생성하기

  1. /src/components/에 tableOfContents.js 파일을 생성한다.
  • 최소한의 소스 코드는 이정도.
import React from 'react';

export default function TableOfContents({ items, currentHeaderUrl }) {
  return items ? (
    <div>
        <p>Table of Contents</p>
      <ul dangerouslySetInnerHTML={{ __html: items }}
      >
      </ul>
    </div>
  ) : null
}
  1. /src/templates/blog-post.js 기존 파일 내용을 수정했다.

TableOfContents 를 import하고, return 내 Layout 위쪽에 추가했다.
