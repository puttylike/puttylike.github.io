데이터 클렌징 작업 - 이름 컬럼 (by SQL 정규식)

모 테이블에 이름이 들어가는 컬럼인데, 필수값 설정도 안 돼 있고 그렇다고 값 입력 받을 때 한글만 또는 영문만 제대로 입력하게끔 화면에서 제어하는 로직은 또 없었나 보다. 그렇다고 이게 뭔가 되게 중요한 정보로 잘 관리가 되던 거도 아닌 거 같다. 그래서 오류율이 상당히 높았다.

클렌징 대상 데이터 유형은 다음과 같았다. 내 기억에는 분명... 분명히 11가지가 넘었는데, 쿼리 보고 와서 더 세분화하여 정리해 놓아야 겠다 ㅠㅜ [*** 09.25 기준 순서 정리 전ㅠ]

  1. 숫자만 있는 경우 => 알 수 없음
  2. 공백+숫자만 있는 경우 => 알 수 없음
  3. 공백이 포함되는 경우 => 일단, 공백 제거
  4. 특수문자+숫자인 경우 => 알 수 없음
  5. 특수문자+한글인 경우 => 특수문자 제거
  6. 한글+숫자인 경우 => 대체로 한글로
  7. 한글+영어인 경우 => 대체로 한글로
  8. 자음 또는 모음 그 자체가 포함되는 경우 => 일단, 자음 제거
  9. 알파벳만 있는데, 글자 길이 2 이하인 경우 => 알 수 없음
  10. 클렌징 대상 건 한글 중에서 누가 봐도 사람 이름 아닌 경우 => 알 수 없음
  11. [아마도, 마지막 작업] 누가 봐도 오타로 보이는 경우 => 건 별로 수정
  12. 알파벳 2개 이하 => 알 수 없음

오라클 DB에 맞게 sql 정규식을 활용해서 쉬운 순서대로 update 쿼리를 작성했다. 누가 봐도 ㅣ를 ;로 쓴 거 같은 오타는 건 별로 수정했다...

데이터를 일괄적으로 알 수 없다고 표기하는 건 좀 아닌 거 같아서 ㅎㅎ... 개명하면 그만 아니냐, 라고 말하던 사람도 있었지만, 데이터 유실률이 좀 아깝게 느껴졌다. 화면 쪽 수정되면서 앞으로는 잘 관리할 데이터이고, 이게 다른 업무 쪽에서도 쓰이고, 필요성이 더 증대될 수 있는 데이터라고 하니까 최대한 살려보고 싶었다.

테스트 DB에서 조회 및 수정 확인까지 다 하다 보니4시간 정도 소요됐다. 이후, 담당자에게 엑셀로 정리하여 확인 요청을 드렸다... 두근두근 하다. 대체 이게 몇 건이야 ㅠ 필수값이 아니면 사람들은 입력하는 게 너무 귀찮은가 보다. 그래서 그런 바쁜(...) 상황은 이해하지만, 데이터 관리 및 활용을 위해 더욱더 제어 로직을 잘 심어줘야 된다고 생각했다.
