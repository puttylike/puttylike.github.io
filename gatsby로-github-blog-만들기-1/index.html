깃허브로 블로그 만들기 - 1

(1) github 접속

  1. 회원 가입 (https://github.com/)
  2. github 로그인
  3. Repositories 생성

ex) Repository name : puttylike.github.io
=> 이 경우, 블로그 주소는 https://puttylike.github.io/ 가 된다.

(2) 개발환경 설정 : 맨 위의 갯츠비 튜토리얼 참고

  1. Node.js 설치 / nvm (노드 버전 관리자) 설치
  2. Git 설치 (git for windows : https://gitforwindows.org/)
  3. Gatsby CLI 설치 : npm을 통해 전역적으로 설치한다.

cmd> npm install -g gatsby-cli
cf. cmd> gatsby --help

(3) 갯츠비로 웹사이트 만들기 : git bash 실행

  1. 새로운 사이트 만들기 : 원하는 경로에서 다음의 명령어를 실행한다.

$ cd /C/Users/delta/
$ gatsby new [SITE_DIRECTORY_NAME] [URL_OF_STARTER_GITHUB_REPO]
ex) gatsby new puttylike.github.io https://github.com/gatsbyjs/gatsby-starter-hello-world
$ cd puttylike.github.io
$ gatsby develop
cf. gatsby develop 후, 로컬에서 사이트 확인 (http://localhost:8000/)
cf. ctrl+c를 누르면 빠져 나올 수 있다.
cf. GraphiQL (http://localhost:8000/___graphql)

  1. 깃허브 배포 : gh-pages를 설치해 배포 관리 중이다.

$ cd /C/Users/delta/puttylike.github.io
$ npm install gh-pages --save-dev

  1. package.json 파일 scripts 안쪽에 아래 1줄을 추가한다.

"deploy": "gatsby build && gh-pages -d public -b master",

  1. gatsby-config.js 파일 module.exports 안에 siteMetadata 안에 url을 다음과 같이 추가한다.

pathPrefix: "/puttylike.github.io", // my blog url : repo name

** 이거 한 번 하고 나서는, 이거 3개 정도 자주 쓰는 중이다...
$ npm run develop // 로컬
$ npm run build // 빌드
$ npm run deploy // 블로그 배포

  1. 블로그 배포가 완료되면, repository name으로 설정한 웹사이트 주소에 접속해 보면 된다.
