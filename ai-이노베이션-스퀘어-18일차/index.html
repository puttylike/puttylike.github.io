Tensorflow 2.0

TF 1 버전 대에 비해 user-friendly하며, code 직관성을 높이고 디버깅을 쉽게 할 수 있는 eager execution 적용되었다.

TensorFlow 2.0 주요 특징

(1) eager execution 즉시실행모드 : session 이 필요 없고, python처럼 그때그때 실행된다.
lazy evaluation 형태로 코드를 짤 수 있긴 하나 그닥 쓰진 않는다.
= 프로그램 중간 중간 print 문 등을 넣어 언제든 동작 방식을 확인할 수 있다 = interactive 방식
​ cf. 1 버전대는 lazy evaluation 방식 = complexity 증가 performance 증가 (소스 코드가 복잡하나 속도가 빠르다)
= batch script 이런 방식은 perform 증가하나 debugging이 어렵다 = none-interactive 방식 = 대화형이 아니다 쭉 진행하면 끝

(2) keras = google에서 유일한 api로 선정했다. (tf.keras 사용 권장)

(3) api cleanup = tf.placeholder 삭제, tf.contrib 삭제 등
(placeholder : session 내에서 외부 입력 받을 때 사용했으나 session이 없어져서 같이 소멸한 것으로 보임.)
cf. contrib = RNN / LSTM / GRU 사용되는데, 2 버전으로 오면서 삭제돼 새로 짜야된다.

  • 상수 - tf.constant : tf는 기본 데이터 타입이 tensor. 2 버전대에서는 sess.run 대신 .numpy() 붙이면 값이 나온다.
  • 변수 - tf.Variable : 2 버전대에서는 변수 정의 시 자동 초기화된다. 초기화 소스 코드가 필요 없다.
  • placeholder 삭제 : 값을 리턴해 주니까 2 버전 대에서는 함수로 표현하면 된다.

Keras는 model 바탕으로 architecture 설계한다.

model = 입력층, 은닉층, 출력층 각 layer를 포함하는 프레임 워크로,
model 생성 후, 각 layer를 add하면 된다.
(cf. java gui 설계 = canvas + textbox + label ...)

model = Sequential() # 모델 생성 => layer 코드 순서대로 add된다
model.add(...) # 층 추가
model.add(...) # 층 추가

model.compile(...) # 손실함수 지정
                   # 옵티마이저 지정
model.summary() # ! 생략 가능

model.fit(...) # 학습 진행

model.evaluate()
model.predict()

model.save()
load_model()

Sequential API는 케라스의 함수형 API(Keras Functional API) 의 일종이다.

입력 ~ 출력까지 데이터 가는 거는 feed forward => sequential

  • ML = 데이터 정의 / w, b초기화 / y, loss 계산식 / 학습 for
  • KERAS ML = data 정의 / model 정의 아키텍처 설계 / model.compile() / model.fit()

​

model.add(Flatten(input_shape=(1,))) # 입력층
model.add(Dense(1, activation='linear')) # 출력층

# 이렇게 1줄로 표현해도 된다.
model.add(Dense(1, input_shape=(1,), activation='linear'))

처음에는 Flatten을 쓰나, 앞뒤로 연결된 경우에는 Dense를 쓴다.

model.add(Dense(1, activation='linear')) => 1 출력에서 나가는 거를 말한다.
입력층과 붙어있는 애만 input_shape을 쓴다. add 앞뒤로 연결한다는 의미다.

Dense 안에 W, b가 들어 있다. 이렇게 keras는 자동으로 param을 만들어준다.
add 할 때마다 w, b가 자동으로 초기화된다.

  • compile = loss func, learning rate, SGD 미분 정의

model.compile 이후 보통 model.summary를 쓴다 w = (1,1) b = (1) => parma 수 2개

  • fit = for W, b update

예제2 index label prediction list by list comprehension

real_prediction_val = model.predict(x_data)
logical_prediction_val = (real_prediction_val > 0.5) # [[True], [False], ... ]

logical_prediction_val = logical_prediction_val.flatten().astype('float32') # 2차원 -> 1차원
label = t_data.flatten()
comp_result = np.equal(label, logical_prediction_val)

index_label_prediction_list = [ [idx, label[idx], logical_prediction_val[idx] ]  for idx, value in enumerate(comp_result)  if value == False ]
