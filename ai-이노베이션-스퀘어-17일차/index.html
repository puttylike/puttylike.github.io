완전 연결층(Flatten)

​ Tensorflow : node 정의 / 세션 실행으로 나뉜다.

CNN : filter 설정, 초기화 => 노드 - conv2d, maxpool / sess.run() feeddict ...으로 나뉜다.

​filter는 데이터가 이동하는 통로로, feature extraction 역할을 한다.
필터가 32개라는 건 어떤 하나의 데이터에 대해 32개의 특징을 뽑겠다는 의미다.

ex. 3x3 filter 64개이면, 3x3x64 => 64 feature extraction & 32x64 total data

ex. 1st conv에서 3x3 filter 32개이고, 2nd conv에서 4x4 filter 64개이면,
=> 앞에 32개 중 1개의 data에 대한 64개의 특징을 뽑았다고 볼 수 있다.
=> A2는 tensor로, 1st conv에서 max_pool을 거쳤다면 14x14 32개가 된다.

출력층의 목적은 정답과 비교하는 것이다.

일반 신경망 NN에서는 채널이 없고, node 다 나열해서 FC에서 vector로 나왔다. 그래서 NN에서는 정답이 vector, y도 vector였다.

그런데 CNN에서 A2는 tensor인데, 출력층은 one-hot encoding 을 취해야 해서 vecotr이다.
=> tensor 32개를 일렬로 펴서 vector로 만들어야 한다. 이를 완전연결층 Fratten에서 수행한다. = vector로 만들어줘야만 정답과 비교가 가능하기 때문에 필요하다 ★ 완전 연결층

필터의 크기, 갯수 => 숫자
1개 이상의 convolution 층 => 숫자

숫자로 표현될 수 있다는 건 임의로 바꿀 수 있단 거고, 이에 따라 성능이 바뀔 거란 거다.

예제5-1 conv 3층으로, 대신 필터는 3x3

정확도는 0.9922 -> 0.9924가 됐다

conv 1개일 땐 output이 28x28 1개 였는데, conv 3개가 되면서 output이 4x4 128개가 된다. = trade off

3번째 층 max_pool 통해 7x7 -> 4x4 되었다 : 부족한 부분을 채워 넣는다 SAME / VALID 내림으로 3x3 된다

특징 압축할 때마다 크기가 줄어든다

파라미터 갯수

Param 수 = 가중치 수 + bias 합계 수 (* filter : W 집합 , filter 당 bias 1개)

위 예제 5-1에서는, 1st : 3x3x32 + 32
2nd : 3x3x64 + 64
3rd : 3x3x128 + 128 4th : fC 뒤에도 (4x4x128,10) + 10 ​=> 320+640+1280+20480+10 = 22730 = param 수

예제5-2 2nd filter size를 3x3에서 5x5로 변경

3x3 32 conv 패딩있음 2x2 max pool / 5x5 64 conv 패딩있음 2x2 max pool / 3x3 128 conv 패딩있음 2x2 max pool

28x28 1 -> 14x14 -> 7x7 -> 4x4 128 3x3x32 + 32 =320 5x5x64 + 64=1664 3x3x128 + 128=1280 4x4x128x10 + 10=20480+10 => 23754 = param 수

Accuracy = 0.9912 가 나온다. 얼추 비슷하다.

data type : flatten 이전 tensor | flatter 이후 vector
functionality : feature extraction | classifier 분류기

=> CNN tensor 기반 feature 추출해서 정확도가 높다
